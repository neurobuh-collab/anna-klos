# Заявки с сайта → Telegram-бот

Форма-заявка на сайте отправляет данные в бот через маленький посредник (релей)
на **Google Apps Script**. Токен бота хранится в Google и **не виден** в публичном коде сайта.

```
Сайт (форма)  →  Google Apps Script (/exec)  →  Telegram Bot API  →  ваш чат
```

## Шаг 1. Токен бота

Токен вы уже получили от [@BotFather](https://t.me/BotFather) при создании бота
(строка вида `123456789:AAE...`). Если нужен новый: `/newbot` в @BotFather.

## Шаг 2. Узнать chat_id (куда падают заявки)

1. Напишите своему боту любое сообщение (например, «привет») в личку —
   или добавьте бота в группу и напишите там.
2. Откройте в браузере (подставьте свой токен):
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates`
3. Найдите в ответе `"chat":{"id": ... }`. Это и есть `chat_id`
   (для лички — положительное число, для группы — отрицательное, начинается с `-100`).

## Шаг 3. Развернуть релей

1. Откройте <https://script.google.com> → **Новый проект**.
2. Удалите пример кода, вставьте содержимое [`telegram-relay.gs`](telegram-relay.gs).
3. Впишите `BOT_TOKEN` и `CHAT_ID` из шагов 1–2.
4. **Развернуть → Новое развёртывание → тип «Веб-приложение»**:
   - Запуск от имени: **Я**
   - Доступ: **Все (Anyone)**
5. Подтвердите доступ Google (кнопка «Разрешить»).
6. Скопируйте **URL веб-приложения** — заканчивается на `/exec`.

Проверка: откройте этот `/exec`-URL в браузере — должно показать `OK`.

## Шаг 4. Подключить к сайту

Пришлите `/exec`-URL разработчику (или впишите его сами в `index.html`
вместо `__RELAY_URL__` в функции `submitForm`). После этого форма начнёт
доставлять заявки в бот.

## Безопасность

- Токен живёт только в Apps Script, не в репозитории.
- В форме есть скрытое поле-ловушка (honeypot) — отсекает часть спам-ботов.
- Если пойдёт спам на `/exec`, можно пересоздать развёртывание (сменить URL)
  или добавить в релей проверку секретного поля.
